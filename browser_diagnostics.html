🛠️ Troubleshooting Guide

If semantic_features contains "noisy" items:

  1. Check if items have these suspicious patterns: "Retry", "Quick Action", "Session Handler" etc.
  2. Check item names for PK\u0003\u0004 (ZIP binary prefix)
  3. If noisy items exist → Backend filtering failed (Phase 1B issue)
  4. If they're filtered out → Frontend filtering working correctly

If documents field is present:

  1. Check optional_intelligence.semantic_matches[*].documents
  2. If present → Sanitization failed (Phase 2E issue or pre-Phase 2E record)
  3. If absent → Sanitization working correctly

If audit_id or generated_at doesn't match database:

  1. Frontend might be showing old audit record
  2. Check Redis cache or browser LocalStorage
  3. Run fresh audit to regenerate
